systemd

  • 日志查看

    • journalctl

      • 查看所有日志（系统和进程）

      journalctl -k

      • 查看内核日志（不显示应用日志）

      journalctl -b [-0]

      • 查看系统本次启动的日志

      journalctl -b -1

      • 查看上一次启动的日志

      + - journalctl --since ["2018-09-07 21:10:00",yesterday,"20 min ago",09:00] --until ["2018-09-07 21:12:00",'1 hour ago"]

      • 查看指定时间的日志

      journalctl -n 20

      • 显示尾部指定行数的日志

      journalctl -f

      • 实时滚动显示最新日志

      journalctl /usr/lib/systemd/systemd

      • 查看指定服务的日志

      journalctl _PID=1

      • 查看指定进程的日志

      journalctl /usr/bin/bash

      • 查看某个路径的脚本的日志

      journalctl _UID=33 --since today

      • 查看指定用户的日志

      journalctl -u php-fpm.service -u nginx.service [-f , --since today]

      • 查看指定一个或多个 Unit 的日志

      journalctl -p err -b -o json

      • 查看指定优先级（及其以上级别）的日志，共有8级 # 0: emerg # 1: alert # 2: crit # 3: err # 4: warning # 5: notice # 6: info # 7: debug

      journalctl --disk-usage

      • 显示日志占据的硬盘空间

      journalctl --vacuum-size=1G

      • 指定日志文件占据的最大空间

      journalctl --vacuum-time=1years

      • 指定日志文件保存多久
  • 概念
    • 系统管理守护进程
      • 取代 initd
      • 并发启动
      • 接管系统和应用日志
    • 管理资源分类12种
      • Service unit：系统服务
      • Target unit：多个 Unit 构成的一个组 
      • Slice Unit：进程组
      • Device Unit：硬件设备
      • Timer Unit：定时器
      • Scope Unit：非Systemd 启动的外部进程 
      • Mount Unit：文件系统的挂载点
      • Socket Unit：进程间通信的 socket
      • Swap Unit：swap 文件
      • Path Unit：文件或路径
      • Snapshot Unit：Systemd 快照，可以切回某个快照
  • 配置
    • 规则
      • 读取目录/etc/systemd/system/ 会覆盖/usr/lib/systemd/system/下相同的service配置
      • 大小写敏感
      • 键值对的等号两侧不能有空格
    • [Unit]
      • 每一个区块，定义元数据及与其它Unit的关系
      • Requires
        • 强依赖，当前 Unit 依赖的其他 Unit,会去启动依赖项
      • Wants
        • 弱依赖
      • BindsTo
        • 它指定的 Unit 如果退出，会导致当前 Unit 停止运行
      • Before
        • 指定Unit在当前 Unit 之后启动
      • After
        • 指定Unit在当前 Unit 之前启动
      • Conflicts
        • 指定的 Unit 不能与当前 Unit 同时运行
      • Condition
        • 当前 Unit 运行必须满足的条件，否则不会运行
      • Assert
        • 当前 Unit 运行必须满足的条件，否则会报启动失败
    • [Install]
      • 最后一个区块，用来定义如何启动，以及是否开机启动
      • WantedBy
        • 表示该服务所在的Target
        • systemctl enable 时符号链接会放入/etc/systemd/system目录下面以 Target 名 + .wants后缀构成的子目录中
      • RequiredBy
        • 该服务依赖的Target
        • systemctl enable 符号链接会放入/etc/systemd/system目录下面以 Target 名 + .required后缀构成的子目录中
      • Alias
        • 当前 Unit 可用于启动的别名
      • Also
        • 当前 Unit 激活（enable）时，会被同时激活的其他 Unit
    • [Service]
      • 只有 Service 类型的 Unit 才有这个区块
      • Type
        • Type=simple
          • 默认值，执行ExecStart指定的命令，启动主进程，单进程前台运行
        • Type=forking
          • 以 fork 方式从父进程创建子进程，创建后父进程会立即退出，多进程
        • Type=oneshot
          • 一次性进程，Systemd 会等当前服务退出，再继续往下执行
        • Type=dbus
          • 当前服务通过D-Bus启动
        • Type=notify
          • 当前服务启动完毕，会通知Systemd，再继续往下执行
        • Type=idle
          • 若有其他任务执行完毕，当前服务才会运行
      • ExecStart
        • 启动当前服务的命令
      • ExecStartPre
        • 启动当前服务之前执行的命令
      • ExecStartPost
        • 启动当前服务之后执行的命令
      • ExecReload
        • 重启当前服务时执行的命令
      • ExecStop
        • 停止当前服务时执行的命令
      • ExecStopPost
        • 停止当其服务之后执行的命令
      • KillMode
        • 定义 Systemd 如何停止  服务
        • KillMode=control-group
          • 默认值）：当前控制组里面的所有子进程，都会被杀掉
        • KillMode=process
          • 只杀主进程,子进程打开的session 仍然保持连接
        • KillMode=mixed
          • 主进程将收到 SIGTERM 信号(要求程序自己退出)，子进程收到 SIGKILL 信号(进程终止不了，再试这个 kill -9 ????)
        • KillMode=none
          • 没有进程会被杀掉，只是执行服务的 stop 命令
      • RestartSec
        • 表示 Systemd 重启服务之前，需要等待的秒数.
      • Restart
        • 定义何种情况 Systemd 会自动重启当前服务
        • no
          • （默认值）：退出后不会重启
        • always
          • 不管是什么退出原因，总是重启
        • on-success
          • 只有正常退出时（退出状态码为0），才会重启
        • on-failure
          • 非正常退出时（退出状态码非0），包括被信号终止和超时，才会重启 -----on-abnormal只有被信号终止和超时，才会重启
        • on-abnormal
          • 只有被信号终止和超时，才会重启
        • on-abort
          • 只有在收到没有捕捉到的信号终止时，才会重启
        • on-watchdog
          • 超时退出，才会重启
      • TimeoutSec
        • 定义 Systemd 停止当前服务之前等待的秒数
      • Environment
        • 指定环境变量
      • WorkingDirectory
        • 工作目录
      • PrivateTmp
        • 程序在tmp目录下私有，安全
  • 命令
    • Service unit：系统服务
      • 列出相应Unit
        • systemctl list-units --state=failed
          • 查看失败状态
        • systemctl list-units
          • 列出正在运行的 Unit
        • systemctl list-units --all
          • 列出所有Unit，包括没有找到配置文件的或者启动失败的
        • systemctl list-units --all --state=inactive
          • 列出所有没有运行的 Unit
        • systemctl list-units --type=service 列出所有正在运行的、类型为 service 的 Unit
      • 查看Unit状态
        • systemctl status
          • 显示系统状态
        • sysystemctl status|is-active|is-failed|is-enabled  syslog-ng
          • 显示单个 Unit 的状态
        • systemctl -H root@rhel7.example.com status httpd.service
          • 显示远程主机的某个 Unit 的状态
        • systemctl list-dependencies nginx.service
          • 列出一个 Unit 的所有依赖
        • + - systemctl cat nginx
          • 查看某个service配置文件
      • 管理Unit
        • systemctl stop|start|restart|status|enable|disable|show  nginx
        • + - systemctl daemon-reload
          • 重载修改过的配置 systemctl show -p CPUShares httpd.service显示某个 Unit 的指定属性的值
        • systemctl show -p CPUShares httpd.service
          • 显示某个 Unit 的指定属性的值
        • systemctl set-property httpd.service CPUShares=500
          • 设置某个 Unit 的指定属性
    • Target unit：多个 Unit 构成的一个组  
      • systemctl get-default
        • Systemd 默认的启动 Target ，runlevel
      • systemctl list-units --all  --type=target
        • 查看系统有哪些target
      • systemctl list-dependencies multi-user.target
        • 查看target有哪些内容
      • systemctl isolate shutdown.target
        • 切换到另一个 target
    • 系统管理
      • systemctl
        • systemctl reboot|poweroff|rescue单用户|halt
        • systemctl show-environment|set-environment NAME=VALUE|unset-environment
          • 设置系统变量
        • systemctl list-unit-files
      • systemd-analyze
        • 查看启动耗时
        • systemd-analyze critical-chain
          • 显示瀑布状的启动过程流
        • systemd-analyze blame
          • 查看每个服务的启动耗时
      • loginctl
        • loginctl list-sessions |list-users|show-user ruanyf
      • timedatectl
        • timedatectl list-timezones|set-timezone America/New_York|set-time YYYY-MM-DD| set-time HH:MM:SS
      • localectl
        • set-locale LANG=en_GB.utf8|set-keymap en_GB
